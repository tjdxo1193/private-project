# 과제 2: 대용량 파일 목록 UI (5,000개 이상)

대규모 파일/폴더를 효율적으로 렌더링하고 관리하는 UI를 구현합니다.

## 요구사항 요약

### 1. Mock 데이터 생성

- 최소 5,000개 이상의 파일/폴더
- 트리형 구조 (3~4 depth)
- 메타데이터 포함 (size, type, modifiedAt 등)
- 다양한 파일 타입 (이미지, 문서, 동영상, 압축파일 등)
- 실제 파일이 아니라 목록 형태로 표시만 되면 됩니다.

### 2. 핵심 기능

#### 2.1. 폴더 트리
- 폴더 접기/펼치기 기능
- Breadcrumb 네비게이션
- 현재 위치 표시

#### 2.2. 가상 스크롤 (필수)
- **플랫 리스트 모드**: 현재 폴더 내의 5,000개+ 항목을 가상 스크롤로 표시
- **트리 모드**: 폴더를 펼쳤을 때 보이는 모든 항목(부모+자식)을 가상 스크롤로 표시
- **성능 목표**: 10,000개 항목에서도 **60fps 스크롤** 유지

#### 2.3. 다중 선택
- `Ctrl/Cmd + 클릭`으로 다중 선택
- `Shift + 클릭`으로 범위 선택
- 선택된 항목 수 표시
- 선택된 항목의 총 크기 표시
- (선택사항) 선택된 항목 삭제 또는 이동 기능

#### 2.4. 드래그 앤 드롭
- 파일/폴더를 다른 폴더로 드래그하여 이동 (상태만 변경, 실제 파일 시스템 변경 불필요)
- 시각적 피드백 제공 (드롭 가능 영역 하이라이트)
- 드래그 중인 항목의 개수 표시

#### 2.5. 기타 기능
- 파일 타입 필터 (이미지, 문서, 동영상 등)
- 검색 & 하이라이팅
- 정렬 옵션 (이름, 크기, 수정일)
- 리스트/그리드 뷰 전환

### 3. Mock API 필요 여부

⚠️ 이 과제는 백엔드 API가 필요하지 않습니다.

모든 데이터는 프론트엔드에서 생성한 Mock 데이터를 사용합니다.

## 기술 스택

- React + TypeScript
- Virtual scroll: react-window, react-virtualized 등
- 성능 측정 결과 제출

## 평가 기준

- 5,000개 이상의 항목을 부드럽게 렌더링하는가?
- 가상 스크롤이 정상적으로 동작하는가?
- 폴더 트리 구조가 직관적으로 표현되는가?
- 검색, 필터링, 정렬이 빠르게 동작하는가?
- 드래그 앤 드롭이 자연스럽게 동작하는가?
- TypeScript를 활용한 타입 안정성
- 성능 최적화 노력 (렌더링 속도, 메모리 사용량 등)
- 코드 구조와 가독성
- 사용자 경험(UX)
